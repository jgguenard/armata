Le but des macro c'est de clearer l'interface, et de jouer avec une ou 2 barres max et de ne pas se faire chier avec 40 skills.

L'ordre dans lequel on monte la macro est très importante parce que dans rift la macro stop après avoir fait la première skills qui étais disponible.

Alors une macro normal:

Tout dabord on fait apparaitre le logo dune ranged skill. Pourquoi un #show x ranged ability? Ceci donne le logo du skill, mais ranged pour quil s'"allume" lorsque vous etes dans le range pour le faire.

Ensuite les skills réactionnaires, genre après un block ou un counter strike. Bien sur il faut mettre la les skills qu'on se fou de lancer des qu'ils sont up. Si vous avez un disarm de 5 sec par exemple, c'est peut être mieux de le binder sur une autre touche pour vraiment l'utiliser au bon moment.

Par la suite, les skills avec un bon cooldown

Ensuite, les skills corp a corp spammable.

Apres, les skills ranged avec cool down.

Pour finir avec les skills ranged spammable...


example de macro

#show Flamespear (ranged ability)
cast Furious Rage (reactive ability)
cast Face Slam (melee ability with a cooldown such as 6 seconds)
cast Reckless Strike (Melee Range Spamable)
cast Stonespear (ranged ability with cooldown)
cast Flamespear (ranged spamable)


Example tu chase un gars, tu spamm la macro, ca devrait aller a ton premier skill de ranged qui est up soit celui avec cool down ( Stonespear ) la tu stun le gars, tu continue a la balancer des Flamespear pendant que tu te rapproche, aussitot que tu est en range melee, ca va switcher automatiquement, pis tu va le frapper avec ton premier skill de la list qui est up (Face Slam). Ensuite ça va aller directement a Reckless Strike et des que Furious charge est up, juste a pitonner sur le même bouton et le cycle va rouler comme dans du beurre. Alors très important de mettre les skills dans le bon ordre.

Ceci dit, moi perso jme fait une macro single target, et une macro AoE, alors juste avec 2 slot jai deja la pluparts de mes rotations de combat, les autre slot ca sert pour mes skills unique, les gros heals, les skills durgence, les skills hors global cool down...
...

#show Healing Grace

Ça dépend des préférences de chacun .

Moving on. Let's say you're a healer. You've got three heal spells, one with a low cast time and low heal amount, one with a little more cast time and a little more healing, and your big heal with a long cast time.
Three action bar buttons? And you have three souls worth of spells to organize??? No thank you.

/cast @mouseover [shift] Big Heal
/cast @mouseover [ctrl] Mid Heal
/cast @mouseover Baby Heal

ONE action bar button.

The below 3 types of macros are extremely useful. If you are able to master these 3 combos for your class then you can get rid of a crap ton of buttons off your hot bar. Enjoy!

Alternate Target (Mouse Over) Macro
Code:

#show Spark (ranged taunt)
stopcasting (this will stop you if you are in the process of casting)
cast @mouseovertarget Spark (ranged taunt)

The above macro will taunt your mouse over target and then run through all your available attacks on it. You could replace this with your crowd control like Transmogrify so that you can CC an add without changing your target.
EDIT: This Alternate Target Macro can also be used by healers / buffers. Just put your heals in here and you can mouse over the portraits of your group / raid members. This removes you having to click on a person. This is great for you keeping your main tank targeted while healing the off tank or someone else needing it.

Self Purge Macro
Code:

#show Spell Breaker (ally/self dispell)
target MaddBomber83 (will target you if you put in your name)
cast Spell Breaker (ally/self Dispell)
targetlasttarget

The above macro will cast your dispel (or heal, or whatever else you want) on you and then swap back to your current target.

Here is a Guide I found that will better explain RIFT Macro's:

Macros

A macro in the context of video games refers to a set of instructions that can be issued together with one input or Â“hotkey.Â” Rift supports macroing via an in-game menu. Opening that menu brings up a panel with a list of macro slots on the left, and the macro fields on the right. You can give your macro a name and select an icon for it. You can also have a macro inherit the icon and cooldown timer of some ability using the following syntax:

#show ability

e.g. #show Commander´s Order will cause the macro to inherit the icon and ability icon of the skill called Â“Commander´s OrderÂ”


Macro Commands

An important component of macros is the ability to issue game commands via text input. A macro generally saves one or more of these commands for future and repeated use. Some the game commands available are below.

* /saveequip followed by an index number (e.g. Â“/saveequip 1?)
o This saves the currently equipped set of equipment to the specified index.
* /loadequip followed by an index number (e.g. Â“/loadequip 1?)
o This loads the saved set of equipment from the specified index.
* /cast followed by an ability name (e.g. Â“/cast Motif of TenacityÂ”)
o This casts the specified ability. You can specify a target for that ability using special targeting syntax (e.g. /cast @mouseover Commander´s Order) .
* /stopcasting
o This tells the game engine to stop the current action and is a necessary intermediate step between chained macro commands.
* /use item
o This uses the item named (e.g. /use Blessed Tome)
* /help
o This lists the Rift recognized / commands.

Syntax

When listing commands in the macro interface, you do not need the leading slash, so rather than /cast, you would just use cast. The following illustrates:

#show Commander's Order
cast @mouseover Commander's Order
